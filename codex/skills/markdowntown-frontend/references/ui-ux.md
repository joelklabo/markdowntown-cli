# UX patterns distilled

## Primary flow language
- Use consistent labels: "Scan", "Workbench", "Export".
- Prefer one primary CTA per surface.

## Workbench
- Keep key actions (save/export/copy) visible and uncluttered.
- Ensure tabs remain usable on mobile.

## Atlas Simulator
- Make scan results and next-step CTAs prominent.
- Summarize scan status before asking users to act.

## Library / Browse
- Emphasize quality signals (views, copies, recency) and clear CTA hierarchy.
