# Content style + terminology

## Core references
- `docs/ux/microcopy-guidelines.md` (terminology + CTA hierarchy)
- `docs/design/content-design-system.md` (voice, labels, strings)
- `src/lib/strings.ts` (centralized UI strings)

## Terminology rules
- **agents.md** = exported output
- **AGENTS.md** = repo instruction file
- Prefer “Scan”, “Workbench”, “Library”, “Translate”, “Docs” in labels

## Formatting
- Sentence case for headings and buttons.
- Keep helper text to 1–2 short sentences.
- One primary CTA per surface; secondary and tertiary are optional.

## Copy tone
- Direct, calm, action-oriented.
- Explain what happens next; avoid internal jargon.
