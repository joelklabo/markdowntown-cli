# Skill writing checklist

Use this checklist before committing a new SKILL.md.

- [ ] Clear one-line description of the skill.
- [ ] Trigger phrases listed and specific.
- [ ] Steps are numbered and actionable.
- [ ] Guardrails include safety and repo constraints.
- [ ] References link to repo docs or source.
- [ ] Tone is concise and technical.
- [ ] No secrets, tokens, or user data included.
- [ ] Commands match project scripts and conventions.
