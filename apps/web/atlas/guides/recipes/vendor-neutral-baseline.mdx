# Vendor-neutral baseline

<Callout title="Outcome" tone="success">
Instruction sets remain portable across tools and models.
</Callout>

## Steps

1. Use plain-language requirements over tool-specific knobs.
2. Avoid relying on proprietary extensions for critical behavior.
3. Capture expectations as tests and lint rules where possible.

## Example

```md
All code changes must include unit tests for new logic.
```
